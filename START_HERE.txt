🔗 API ENDPOINTS
═════════════════════════════════════════════════════════════════════════════════

NEW ENDPOINT:
  POST /api/predict/symptoms
  ├─ Purpose: Get risk prediction from symptoms + measurements
  ├─ Request: patientId, symptoms[], systolic, diastolic, heartRate, ...
  ├─ Response: prediction (0 or 1), features, symptoms
  └─ Effect: Creates alerts if high-risk

EXISTING ENDPOINTS (Now Enhanced):
  POST /api/measurements       → Save health vitals
  GET  /api/alerts             → Get doctor's alerts
  GET  /api/alerts/:id         → Get specific alert
  PATCH /api/alerts/:id/read   → Mark alert as read

🚀 HOW TO GET STARTED
═════════════════════════════════════════════════════════════════════════════════

STEP 1 - Install Backend:
  $ cd server
  $ npm install
  $ npm start
  ✅ Runs on http://localhost:3000

STEP 2 - Install Frontend:
  $ cd client
  $ npm install
  $ npm run dev
  ✅ Runs on http://localhost:5173

STEP 3 - Test the Flow:
  1. Go to http://localhost:5173/Sympthoms
  2. Click "Quick Health Assessment"
  3. Select symptoms (e.g., "Chest Pain")
  4. Enter measurements (e.g., Systolic: 155)
  5. Click "Get Risk Assessment"
  6. See "HIGH RISK" badge
  ✅ Check another browser window for doctor alerts

STEP 4 - View Doctor Dashboard:
  1. Open doctor account in separate browser/window
  2. Navigate to Doctor Dashboard
  3. Watch real-time alert arrive
  4. See patient symptoms and measurements
  ✅ Mark alert as read

For Detailed Setup: See QUICK_START.md
